# 日签分享卡片信息密度设计

## 目标

解决日签在当天动作较少、尤其是 0/5 完成时正文大面积留白的问题，同时让“今日计划”比普通项目符号更容易扫读。

## 范围

只调整 `shareCheckinCard` 导出的 1080×1440 日签画布及其回归测试；不改变训练数据、完成判定、下载文件名或其他四类分享卡片。

## 布局

1. 保留顶部的训练日、日期与连续打卡信息，但压缩其下方的间距。
2. 将完成进度绘制为独立圆角卡片：左侧显示“今日进度”和已完成/总数，右侧显示百分比与进度条，取代分散的标题和细条。
3. 将“今日计划”改为纵向动作卡片列表。每行包含序号、清晰的完成/待完成状态、动作名和右侧状态文案；动作间有低对比边框或分隔，长名称继续省略。
4. 动作列表从进度卡片后连续向下排布，不再为了填满正文而拉大行距。完成度为零时，列表仍保持明确的视觉层级。
5. 底部标语与水印维持在既有 `layout.footer` 安全区；正文只使用其上方空间，保证不会重叠。

## 自适应规则

- 列表行高固定，动作超过可用高度时沿用现有“另 N 项动作”折叠规则。
- 无动作时显示中性空状态卡片，不绘制空白列表。
- 所有新增文字均使用现有 Canvas 文本测量和省略工具，避免窄空间内溢出。

## 验证

- 新增日签 Canvas mock 回归：进度卡、动作卡均位于正文安全区；第一项动作紧跟进度卡，底部不会侵入页脚。
- 覆盖 0/5 和长动作名称，保留当前文件名与画布尺寸断言。
- 运行全部 Node 分享卡片测试与内联脚本解析检查。
