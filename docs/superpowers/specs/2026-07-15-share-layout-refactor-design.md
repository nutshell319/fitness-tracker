# 分享卡片自适应布局重构 — 设计规格

> 日期：2026-07-15<br>
> 状态：用户已确认设计方向，待审阅实施规格<br>
> 范围：`index.html` 内的热力图、日签、周报、月报与年报导出及其统一预览弹窗

## 目标

消除五类分享图因固定坐标、文本不换行和数据条目数量变化而导致的内容重叠；保持导出图片的比例稳定，并兼顾微信聊天/朋友圈、小红书和微博等场景的清晰度。

不改变训练统计口径、LocalStorage 数据格式、分享入口或 PNG 保存流程。

## 设计决策

采用“按内容使用三套模板”，而不是强制把所有导出图塞进同一比例。

| 模板 | 适用导出图 | 固定导出比例 | 设计理由 |
|---|---|---:|---|
| 竖版成就卡 | 日签、年报 | 3:4（1080 × 1440） | 适合手机信息流；日签强调当日完成，年报承载纵向回顾。 |
| 横版数据报告 | 周报、月报 | 16:9（1600 × 900） | 适合四项统计、趋势和肌肉分布等横向对照数据。 |
| 横版全年热力图 | 热力图 | 16:9（1600 × 900） | 一年最多 53 列，横向空间能保留格子与月份标签的可读性。 |

所有尺寸均为最终 PNG 像素尺寸。绘制代码使用相应的设计缩放系数，避免导出后放大模糊。

## 统一布局内核

Canvas 没有 Flex 或 Grid，因此新增一组纯计算的布局工具；每个绘制器只接收“可用矩形”，不再散落依赖固定的 `y` 坐标。

### 布局契约

1. 每张卡片先创建固定尺寸的画布，并得到统一安全区：外框、标题区、内容区和页脚区。
2. 每个内容区块按顺序测量并绘制，绘制函数返回其实际消耗高度；下一个区块从前一个区块的末端开始。
3. 内容区块只能在其分配的矩形内绘制。任何文字、统计格、热力格或条形图的坐标都不得超出安全区。
4. 布局计算和画图函数分离：前者可用模拟文本测量独立测试，后者只负责调用 Canvas API。

### 新增的通用能力

- `getShareFormat(type)`：返回五类导出图的画布规格、外边距和最小字号。
- `createShareLayout(format)`：生成标题、主体、页脚的安全矩形与纵向游标。
- `fitText` / `wrapText`：根据可用宽度测量文本，支持单行省略和最大行数换行。
- `drawTextBlock`：在给定矩形中绘制并返回实际高度；不允许无边界 `fillText`。
- `drawMetricGrid`：根据宽度、列数和间距自动计算统计卡的矩形。
- `fitHeatmapGrid`：根据可用矩形和实际周数计算格子边长、间隙与标签位置。
- `drawRankedBars`：根据可用高度计算可显示的肌肉群行数与柱状条宽度。
- `drawShareFrame` / `drawShareFooter`：统一背景、边框、品牌水印和底部安全边距。

## 内容优先级与溢出策略

自适应不等于无限缩小字体或无限拉长图片。各模板必须保持固定比例，按下列顺序释放空间：

1. 正常间距缩至模板定义的最小间距。
2. 文本在最小可读字号以上自动换行；标题、动作名等单行字段按可用宽度添加省略号。
3. 以“核心信息优先”的方式压缩次要条目：
   - 日签保留完成进度和尽可能多的动作；超过可显示行数时显示“另 N 项动作”。
   - 周报与月报保留四项统计、趋势图和前 N 个肌肉群；其余合并为“其他”。
   - 年报保留四项统计、全年热力概览、前 N 个肌肉群与三条年度总结；低优先级补充文案不绘制。
   - 热力图保留标题、核心训练统计、月份标签和图例；格子与间隙按 53 周的最坏情况缩放。
4. 若仍无足够空间，隐藏最低优先级的补充文案，而非与其他内容重叠或低于最小字号。

“Top N”和“另 N 项/其他”应基于真实数据数量动态显示；数据量较少时不显示这些提示。

## 各卡片布局

### 日签（3:4）

从上到下为：训练日标识、日期、连续打卡、激励语（最多两行）、完成进度、动作列表、底部标语和水印。动作列表占用剩余主体空间，行高由可用高度计算。动作名称使用单行省略，绝不覆盖底部标语。

### 周报与月报（16:9）

标题、四项统计格、主图（周趋势或当月热力）、肌肉分布、总结和水印依次布局。统计格宽度自动均分；周趋势始终使用 7 个等宽列；月度热力根据实际月历行数计算单元格大小。总结最多两行。

### 年报（3:4）

标题与副标题、四项统计格、全年热力概览、肌肉分布、年度总结、鼓励语和水印依次布局。全年热力固定为 12 个等宽月份槽位，每月内部按周数压缩；肌肉条根据剩余高度决定显示数量。

### 热力图（16:9）

显示当前日历年度，不再从训练开始年份无限延伸到当前日期。画布按照最坏情况 53 周计算网格，以保证跨年或训练周期很长时不会生成超宽图片。标题统计行可在窄宽度下分成两行，图例自动换行或缩至最小间距。

## 统一预览弹窗

预览弹窗改为三行 Grid：标题（自动高度）、图片区域（`minmax(0, 1fr)`）和操作区（自动高度）。

- 外层最大高度使用 `90dvh`，并保留 `90vh` 兼容回退。
- 图片区域必须 `min-height: 0; overflow: auto`，图片使用 `max-width: 100%; max-height: 100%; object-fit: contain`。
- 操作区在窄屏换为全宽双按钮；关闭按钮始终可见。
- 补齐未定义的背景和文字 CSS 变量，预览区域不再出现透明背景或继承颜色偏差。

## 错误处理与兼容性

- 字体测量、Canvas 上下文获取和图片 Data URL 生成失败时，沿用现有 Toast 提示，不展示空白预览。
- 所有布局计算仅使用现有浏览器 Canvas 2D API，不引入外部依赖。
- 对不支持 `roundRect` 的情况使用现有兼容绘制封装，或提供最小圆角回退，确保目标浏览器可导出。

## 可验证性

新增零依赖的 Node 测试，验证纯布局计算而不是 Canvas 像素：

1. 五种导出图返回的比例符合规格。
2. 统计格和热力格的矩形始终位于画布安全区内。
3. 53 周热力图可在 16:9 画布内得到正值格子尺寸与非重叠坐标。
4. 长文本会换行或省略，不会返回超过最大行数的高度。
5. 多达 7 个肌肉群、长动作列表等高数据量会触发“其他/另 N 项”策略，而不超出内容区。

浏览器验证覆盖五类导出图在桌面和 375 px 宽度手机视口中均能完整预览、关闭和保存。
